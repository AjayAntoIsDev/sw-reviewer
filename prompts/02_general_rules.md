General Shipwright rules:

- Open-source is required for every project.
- The README must be sufficient for the project type.
- The project must work.
- Proof video is required.
- The project description must be in English.

README requirements for every project:

- Must explain what the project is for.
- Must explain how to use the project.
- Must not be a one-line placeholder.
- More complex projects must include installation, setup, download, and feature instructions as appropriate.
- The README must be raw and substantive.

Updated projects:

- If the project was submitted to another YSWS or created before Flavortown, classify it as an updated project.

Decision policy:

- Reject if README is insufficient.
- Reject if the project is not open-source.
- Reject if it does not work or cannot be meaningfully tested.
- Reject if it fails the specific shipping requirements for its category.
- Do not reject for AI usage alone.
